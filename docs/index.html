webboot?

@webboot aims to make tofu - trust on first use a bit less scary.

we (and you?) run a network of validator nodes, providing a source of integrity to homepages and webapps.

publishers / developers.

publishers / developers of addressable content can use the command line interface for now.

we are working on a graphical user interface to accompany the cli.

in addition, we are working on integrations for various static page generators like magic, gatsby and vuepress.

cli - coming soon

first, install webboot.

npm install -g webboot

then, go to the directory with your public page and run

webboot

webboot will guide you through the process.

- release time: soon

the webboot app widget allows you to embed the webboot gui into your page.

this allows you to show the webboot gui to every one of your users, even those that did not yet install any of the browser extensions.

see below for available integrations:

  • @magic - release: 22. 2. 2020

    @magic is a static page generator that generates about 10kb of uncompressed html + css + js boilerplate.

    made by the people that made @webboot.

  • gatsby - coming soon

    gatsby is a free and open source framework based on react and helps developers build blazing fast websites and apps.

validators

validators are nodes in the webboot network.

@webboot is undergoing testing and once we are confident about the deployment story,we will decentralize the network, allowing anyone to run a node.
